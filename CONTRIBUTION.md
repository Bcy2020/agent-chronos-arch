# Contributing

Thank you for your interest in Agent Chronos.

This project is currently an open architecture and research effort. Contributions are welcome not only as code, but also as well-defined engineering problems, design analysis, and small validation prototypes.

## What counts as a contribution

You can contribute in three ways:

### 1. Problem proposals
Open an issue if you identify a key engineering problem that must be solved to make this architecture practical.

Examples:
- workflow orchestration
- contract validation
- clone isolation
- experience pool design
- Git-based state management
- rollback and change handling
- evaluation and benchmarking

### 2. Problem analysis
You can improve an existing issue by clarifying:
- why the problem matters
- where it appears in the architecture
- constraints and trade-offs
- possible solution directions

### 3. Prototypes and implementations
You can submit small experiments, scripts, diagrams, or code that help validate one part of the architecture.

## How to contribute

### Open an issue
Please include:
- a clear title
- the problem statement
- why it matters
- the related architecture section (if known)
- possible solution ideas (optional)

### Open a pull request
PRs are welcome for:
- documentation improvements
- problem list updates
- design notes
- prototype code
- tooling experiments

Please keep PRs focused and explain what problem they address.

## What makes a good contribution

A strong contribution is:
- specific rather than vague
- relevant to the architecture
- technically actionable
- respectful of trade-offs
- written clearly enough for others to continue

## Scope

This repository is not only for building software. It is also for identifying, organizing, and solving the hardest engineering questions behind the architecture.

If your contribution helps move the architecture from concept to reality, it is valuable.

## Communication

Please keep discussions constructive, technical, and respectful.

If you are unsure whether something is worth contributing, open an issue first.
