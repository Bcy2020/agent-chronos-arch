# Contributing

感谢你对 Agent Chronos 的关注。

本项目当前仍处于开放架构与研究阶段。我们欢迎的贡献不只包括代码，也包括高质量的工程问题、设计分析，以及用于验证架构可行性的小型原型。

## 什么算贡献

你可以通过以下三种方式参与贡献：

### 1. 提出问题
如果你发现了一个会影响该架构工程落地的关键问题，欢迎提交 issue。

例如：
- 工作流编排
- 契约校验
- 分身隔离
- 经验池设计
- 基于 Git 的状态管理
- 回滚与变更处理
- 评估与基准测试

### 2. 分析问题
你也可以在已有 issue 的基础上补充和完善，例如：
- 说明这个问题为什么重要
- 指出它在架构中的位置
- 分析约束条件与取舍
- 提出可能的解决方向

### 3. 原型与实现
欢迎提交小型实验、脚本、图示或代码，用于验证架构中的某一个部分是否可行。

## 如何贡献

### 提交 issue
请尽量包含以下内容：
- 清晰的标题
- 问题描述
- 为什么这个问题重要
- 相关的架构章节（如果知道）
- 可能的解决思路（可选）

### 提交 pull request
以下内容都欢迎通过 PR 提交：
- 文档改进
- 问题列表更新
- 设计说明
- 原型代码
- 工具化实验

请尽量让每个 PR 聚焦于一个具体问题，并说明它希望解决什么。

## 什么是高质量贡献

一个好的贡献通常应当具备以下特点：
- 具体，而不是泛泛而谈
- 与架构本身相关
- 能推动实际工程落地
- 能体现约束和取舍
- 表述清晰，便于他人继续推进

## 项目范围

这个仓库不只是为了实现一个软件，也用于识别、整理并解决这套架构在工程落地中最关键的问题。

只要你的贡献能帮助这套架构从概念走向现实，它就是有价值的。

## 沟通方式

请保持讨论建设性、技术性与尊重。

如果你不确定某个想法是否值得贡献，欢迎先提交 issue 进行讨论。
