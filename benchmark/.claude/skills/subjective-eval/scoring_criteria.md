# 评分标准定义

本文档定义主观评价的评分维度、权重和换算规则。

## 维度权重分布

### 代码维度（55%权重）

| 维度 | 权重 | 检查任务 | 判断标准 |
|------|------|----------|----------|
| 代码正确性 | 15% | 代码逻辑是否正确实现声明功能？ | 逻辑完整、边界处理正确、无明显错误 |
| 代码可运行性 | 10% | 是否存在语法/运行时错误风险？ | 无语法错误、无未导入依赖、无参数冲突 |
| 代码风格质量 | 10% | 命名、注释、结构是否合格？ | 有docstring、命名语义清晰、结构合理 |
| 边界遵守度 | 10% | 代码是否超出boundary.in_scope范围？ | 代码操作均在in_scope内，未做out_of_scope声明的事 |
| 接口一致性 | 10% | 函数签名是否与JSON声明匹配？ | 参数名、参数类型、返回类型均一致 |

### 结构维度（45%权重）

| 维度 | 权重 | 检查任务 | 判断标准 |
|------|------|----------|----------|
| 需求覆盖完整性 | 15% | 每个FR是否有对应节点且代码实现？ | FR有对应节点路径，代码实现了功能 |
| 分解粒度适当性 | 10% | 分解粒度是否适当？ | 不过粗（一个节点做太多事）、不过细（拆分无意义） |
| 语义停止达成率 | 5% | 叶节点是否因语义原因停止？ | 不仅要看stop_reason字符串，还要对照实际代码验证：宣称"纯函数"的节点必须无`global`声明、无副作用、无I/O；宣称"原子操作"的节点必须仅操作单一数据源且与声明一致 |
| 全局状态守恒 | 10% | 跨层全局操作是否守恒？ | 父节点与子节点的global_state_operations按source_id分组对比，操作类型必须对等；每个operation的source_id必须在对应的data_sources中声明；叶节点代码中的global声明必须与推导结果一致 |
| 可维护性预估 | 5% | 修改该节点是否容易？ | 职责单一、依赖清晰、接口明确 |

## 比例→分数换算表

| 通过比例 | 分数 | 等级 |
|----------|------|------|
| 95-100% | 5 | 优秀 |
| 80-94% | 4 | 良好 |
| 60-79% | 3 | 合格 |
| 40-59% | 2 | 较差 |
| 0-39% | 1 | 不合格 |

---

## 综合评分计算公式

```
overall_score = Σ(dimension_score × weight)

其中：
- dimension_score: 该维度按比例换算的分数（1-5）
- weight: 该维度的权重（见上表）
```

计算示例：
```
代码正确性: 4分 × 15% = 0.60
代码可运行性: 5分 × 10% = 0.50
代码风格质量: 4分 × 10% = 0.40
边界遵守度: 3分 × 10% = 0.30
接口一致性: 5分 × 10% = 0.50
需求覆盖完整性: 5分 × 15% = 0.75
分解粒度适当性: 4分 × 10% = 0.40
语义停止达成率: 3分 × 5% = 0.15
全局状态守恒: 4分 × 10% = 0.40
可维护性预估: 4分 × 5% = 0.20

总分 = 4.20 (满分5.00)
```

## 输出字段定义

### 维度结果字段

| 字段 | 类型 | 说明 |
|------|------|------|
| passed_nodes | int | 通过检查的节点数 |
| failed_nodes | int | 未通过检查的节点数 |
| total_nodes | int | 总节点数 |
| ratio | float | 通过比例 (0-1) |
| score | int | 换算分数 (1-5) |
| weight | float | 该维度权重 |
| weighted_score | float | 加权分数 (score × weight) |
| failed_details | list | 失败节点详情 [{node_id, reason}] |

### 需求覆盖字段

| 字段 | 类型 | 说明 |
|------|------|------|
| covered_requirements | int | 已覆盖需求数 |
| total_requirements | int | 总需求数 |
| ratio | float | 覆盖比例 |
| score | int | 换算分数 |
| weight | float | 权重 |
| weighted_score | float | 加权分数 |
| covered_details | list | 覆盖详情 [{fr_id, node_path}] |

## 评判注意事项

1. **失败节点必须给出原因**：不能只标记失败，必须说明为什么失败
2. **覆盖需求必须给出路径**：不能只标记覆盖，必须给出对应的节点路径
3. **参考声明而非推断**：评判依据是boundary、purpose、inputs/outputs声明，而非猜测意图
4. **区分声明与实际**：
   - boundary.out_of_scope声明但代码实际执行 → 边界遵守度失败
   - 命名暗示某功能但代码实际做其他事 → 代码正确性失败
